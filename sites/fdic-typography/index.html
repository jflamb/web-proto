Penny Prose Style Guide

A reference for the prose component in the FDIC Design System. Add class="prose" to any container to apply these typographic styles to its content automatically. This paragraph uses class="lead" to introduce the page at a slightly larger size (20px) in a secondary color — useful for article summaries and section introductions.

<article class="prose">
  <p class="lead">Your introductory text here.</p>
  <!-- Your content here -->
</article>

Typography

Headings

Headings use Source Sans 3 at weight 600. Space above a heading is 1.5em (relative to the heading's own font size) and space below is 0.5em, so larger headings get proportionally more breathing room. The h1 has no top margin (it’s assumed to be first) and includes a bottom border as a visual anchor.

Stacked headings

When headings appear consecutively (like the h3 and h4 above), the top margin on the second heading collapses to 0.5em so they read as a group rather than drifting apart.

Body text

Body text is set at 18px (1.125rem) with a line-height of 1.5. This combination gives long-form content enough breathing room for sustained reading without feeling loose. The text-wrap: pretty property improves paragraph rag in browsers that support it.

Fine print

Disclaimers, version notes, and legal caveats are everywhere in regulatory content. The <small> element renders text at a reduced size in a secondary color — readable but clearly subordinate to surrounding content.

The prose component is provided as-is under the FDIC Design System and may change in future releases. Refer to the design system changelog for the latest updates.

Inline Elements

Links

Links are underlined in the FDIC brand blue (#1278b0) with a subtle, partially transparent 1px underline at rest. On hover, three properties animate together: the underline thickens to 2px, drops from a 2px to 4px offset, and saturates from 40% to full opacity — creating a clearly perceptible response without being flashy. Visited links shift to a muted purple (#855AA5) that darkens slightly on hover. Here’s a link to the FDIC website and here’s one you may have visited before. External links display a small arrow icon after the text — for example, the FDIC article on Wikipedia.

Accessibility: All links receive a visible :focus-visible outline (2px solid blue, 2px offset) for keyboard users. External-link icons are applied via CSS background-image and are purely decorative — screen readers announce the link text alone. Ensure link text is descriptive; avoid generic labels like “click here” or “read more,” which are meaningless out of context (WCAG 2.4.4).

Bold and emphasis

Use <strong> to mark text with strong importance — content the reader must not skip, such as warnings, key terms in instructions, or critical figures: deposits exceeding $250,000 are not insured. Do not use <strong> for visual styling alone; if text should be bold but carries no extra importance (like a label in a list), use CSS font-weight: 600 instead.

Use <em> for stress emphasis — words that would be vocally stressed when read aloud, changing the meaning of a sentence. Compare “the bank was adequately capitalized” (it was, despite doubts) with “the bank was adequately capitalized” (capitalized specifically, not necessarily liquid). Do not use <em> for titles of works (use <cite>), technical terms (use <dfn> on first use), or foreign phrases (use <i lang="...">).

Bold text renders at weight 600 (semibold) rather than 700, matching the heading weight and avoiding a heavy appearance in body copy. You can combine them for strong emphasis when both importance and vocal stress apply — but this should be rare.

Inline code

Inline <code> elements render in the system monospace stack at a slightly reduced size, with a subtle background and border: font-size: 0.8em. Use it for tag names like <article>, property names like margin-top, or values like 1.5rem.

Abbreviations

The FDIC was established in 1933 to maintain public confidence in the nation’s financial system.[1] Abbreviations with a title attribute get a dotted underline and a help cursor — hover over “FDIC” to see the tooltip. Use <abbr title="…"> for the first occurrence of any acronym on a page.

Accessibility: The title attribute on <abbr> is not announced by most screen readers by default. Always spell out the full term in visible text on first use — for example, “Federal Deposit Insurance Corporation (FDIC)” — so the expansion is available to all users regardless of assistive technology.

Keyboard input

The <kbd> element represents keyboard input. It renders as a raised pill that mimics a physical key: Ctrl + S to save, Ctrl + Shift + F to toggle full-screen mode. The subtle bottom border creates a three-dimensional appearance.

Highlighted text

The <mark> element highlights text with a warm yellow background. When a quarterly review surfaces problems, the audit may flag three critical findings that require immediate action. The highlight is subtle enough for inline use but visible enough to scan for quickly.

Insertions and deletions

The <del> and <ins> elements mark removed and added text, useful for tracking amendments in regulatory documents. For example, the standard insurance limit was raised from deleted: $100,000 to inserted: $250,000 per depositor in 2008.

Accessibility: Screen readers do not consistently announce <del> and <ins> semantics. Include a visually hidden <span class="sr-only"> inside each element (e.g., deleted: or inserted:) so assistive technology conveys the change. This is a Section 508 best practice for revision-tracked content.

Dates and times

The <time> element associates human-readable dates with a machine-readable datetime attribute. When a title attribute is present, it displays the ISO date on hover: the FDIC was established on June 16, 1933. Quarterly Call Reports are due within 30 days of each quarter end — for example, the Q4 2025 report was due by January 30, 2026.

Sample output

The <samp> element represents sample output from a program or system, visually distinct from <code> (source code) and <kbd> (user input). After submitting a Call Report, the system responds with Report CR-2026-Q1 accepted — confirmation ID 7FA3B. Use <samp> for status messages, error output, or any machine-generated text.

Special characters and entities

Financial and regulatory documents rely on a range of special characters that must be encoded as HTML entities. Using the correct entity — rather than a keyboard approximation — ensures proper rendering, searchability, and accessibility:

  • Quotation marks: Use curly quotes (“insured” and ‘insured’) via &ldquo;/&rdquo; and &lsquo;/&rsquo; rather than straight primes (" /'). The right single quote also serves as the typographic apostrophe: don’t, it’s, depositors’.
  • Dashes: Use an em dash (—) via &mdash; for parenthetical breaks, and an en dash (–) via &ndash; for ranges: pages 181–200, fiscal years 2024–2025.
  • Section sign: §363 sales, 12 U.S.C. §1831o — use &sect; for regulatory section references.
  • Daggers: Use † (&dagger;) and ‡ (&Dagger;) as secondary footnote markers when numbered notes are reserved for citations.
  • Ellipsis: Use &hellip; (…) for a proper single-glyph ellipsis rather than three periods.
  • Legal marks: FDIC® insured — use &copy; (©), &reg; (®), and &trade; (™) for legal designations.
  • Math and comparisons: Capital ratios ≥ 4.5%, tolerance of ±0.5%, and per-transaction fees of 15¢ — use &ge;, &le;, &plusmn;, and &cent;.
  • Non-breaking space: Use &nbsp; between quantities and units (12 U.S.C., $250,000 per depositor) to prevent awkward line breaks.

Superscripts and subscripts

Footnote references use superscript links[2] that don’t disrupt line height — this is achieved with line-height: 0 on sup and sub elements, which prevents them from inflating the line box. Subscripts appear in scientific contexts: CO2 levels, H2O quality, and Fe2O3 concentrations all render at the correct baseline without affecting vertical rhythm.

Inline quotations

The <q> element wraps inline quotations with language-appropriate curly quotes automatically. Some CMS editors emit <q> tags instead of manual quote characters. For example, the FDIC’s mission is to maintain stability and public confidence in the nation’s financial system. Nested quotations switch to single quotes: The examiner noted material weakness in internal controls in the final report.

Defining instance

The <dfn> element marks the defining instance of a term — its first or most authoritative explanation. Prompt Corrective Action (PCA) is a framework of supervisory actions tied to a bank’s capital levels. Once a term has been introduced with <dfn>, subsequent uses should appear as plain text or abbreviations.

Variables

The <var> element represents a variable in a mathematical expression or programming context. For example, a bank is considered well-capitalized when its CET1 ratio r ≥ rmin, where rmin = 6.5%. In system configuration, set threshold to the desired trigger value.

Word break opportunities

The <wbr> element suggests optional line-break points within long strings that would otherwise overflow. This is useful for certificate numbers (CERT#00012345-2026-Q1-RC), long URLs, or regulatory reference codes that lack natural break points. The browser will only break at a <wbr> if the line would otherwise overflow.

Block Elements

Blockquotes

Blockquotes use the brand blue as a left border with slightly increased line-height (1.6) and secondary text color for visual distinction:

“Typography is the craft of endowing human language with a durable visual form.” When done well, it is invisible — the reader absorbs the message without noticing the medium.

— Robert Bringhurst, The Elements of Typographic Style

Blockquotes can contain multiple paragraphs, lists, or other block elements. The left border acts as a continuous visual anchor. Use a <cite> element inside the blockquote for attribution — it renders as a block in a smaller secondary style.

Nested blockquotes

Nested blockquotes appear in legal documents, email threads, and forum-style CMS content where authors quote prior rulings that themselves contain quotations. The inner blockquote uses a lighter border to establish visual hierarchy:

The Board of Directors concluded that the institution failed to meet the requirements of 12 U.S.C. §1831o. As noted in the prior examination report:

“The institution’s Tier 1 leverage ratio of 3.2% falls below the 4.0% threshold required for adequate capitalization. Prompt corrective action is warranted.”

The institution was given 30 days to submit a capital restoration plan.

Aside / pull quote

Key fact: Since 1933, no depositor has lost a single penny of FDIC-insured funds.

The <aside> element represents content tangentially related to its surrounding context — pull quotes, sidebar notes, or “related content” blocks. Many CMS editors expose this as a block type. In prose, asides float right at 40% width with a brand-blue left border, creating a magazine-style pull quote. On narrow viewports, they linearize to full width.

Pull quotes are particularly useful in long-form regulatory content to highlight key statistics, policy changes, or notable findings that readers should be able to scan without reading every paragraph. The aside clears after the surrounding paragraphs flow past it.

Code blocks

Fenced code blocks render in a monospace font against a light background with a subtle border. They scroll horizontally if the content overflows:

.prose {
  font-family: var(--fdic-font-family-sans-serif);
  font-size: var(--fdic-font-size-body, 1.125rem);
  line-height: var(--fdic-line-height-body, 1.5);
  max-width: var(--prose-max-width, 65ch);
  text-wrap: pretty;
}

Wrapping code blocks

By default, code blocks scroll horizontally when content overflows. For configuration files, log output, or prose-heavy code, add class="prose-pre-wrap" to enable word wrapping instead:

ERROR 2026-01-15T09:23:41Z [fdic-ingest] Failed to process Call Report CR-2026-Q1 for institution CERT#12345: validation error in Schedule RC-R, line item 43 — reported Tier 1 capital ratio (3.8%) falls below the minimum threshold (4.0%). Report has been flagged for supervisory review. Contact the institution's primary federal regulator for corrective action within 30 calendar days.

Preformatted text (without code)

CMS editors sometimes produce bare <pre> blocks without a nested <code> element — for ASCII tables, formatted addresses, or fixed-width layouts that aren’t source code. These render identically to code blocks:

  FDIC QUARTERLY BANKING PROFILE — Q4 2025
  ═══════════════════════════════════════════

Addresses

The <address> element marks up contact information for the nearest <article> or <body> ancestor. Browser defaults render it in italic, which the prose component overrides to match body text. Use it for office locations, mailing addresses, or institutional contacts:

Federal Deposit Insurance Corporation
550 17th Street NW
Washington, D.C. 20429
(877) 375-3342

Horizontal rules

Horizontal rules provide strong visual breaks between major sections. They use generous vertical margins (3rem above and below) so they don't crowd adjacent content. There's one between each top-level section on this page.

Lists

Unordered lists

Simple unordered lists use tight spacing to keep items feeling cohesive:

  • Standard deposit insurance covers up to $250,000 per depositor.
  • Coverage applies per insured bank, not per account.
  • Joint accounts are insured separately from individual accounts.

Ordered lists

Ordered lists follow the same spacing conventions:

  1. Identify the failed institution and confirm FDIC insurance status.
  2. Determine the applicable insurance limit per ownership category.
  3. Calculate the insured amount for each depositor.
  4. Issue payments to insured depositors, typically within two business days.

Complex list items

When list items contain paragraphs or supporting text, more space appears between siblings than between a bullet and its content. This proximity grouping keeps each item's details visually bound to its heading:

  • Single-ownership accounts.

    Deposits owned by one person in one insured bank are insured up to $250,000. This includes checking accounts, savings accounts, money market deposit accounts, and certificates of deposit.

  • Joint accounts.

    Deposits owned by two or more people at the same insured bank are insured up to $250,000 per co-owner. For example, a joint account owned by two people is insured up to $500,000.

  • Revocable trust accounts.

    Deposits held in a revocable trust (including living trusts and payable-on-death accounts) are insured up to $250,000 per beneficiary, subject to specific conditions and limitations.

Nested lists

Nested lists use proximity grouping: there's more space between sibling items than between a parent item and its nested children, so the hierarchy reads clearly:

  1. Capital adequacy ratios
    • Common Equity Tier 1 (CET1) ratio: minimum 4.5%
    • Tier 1 capital ratio: minimum 6%
    • Total capital ratio: minimum 8%
  2. Leverage requirements
    • Tier 1 leverage ratio: minimum 4%
    • Supplementary leverage ratio (for large banks): minimum 3%
  3. Buffer requirements
    • Capital conservation buffer: 2.5% above minimums
    • Countercyclical buffer: 0–2.5%, set by regulators
    • G-SIB surcharge: 1–3.5%, for systemically important banks

Lettered and Roman numeral lists

Legal and regulatory text frequently uses lettered (type="a") and Roman numeral (type="i") sub-lists for nested clauses. Padding adjusts automatically to accommodate wider markers:

  1. Section 3(a) — Definitions. For the purposes of this regulation:
    1. The term “insured depository institution” means any bank or savings association the deposits of which are insured by the Corporation.
    2. The term “deposit” has the meaning given in section 3(l) of the Federal Deposit Insurance Act.
      1. This includes checking accounts, savings accounts, and certificates of deposit.
      2. This excludes investment products such as stocks, bonds, and mutual funds.
      3. Trust arrangements are evaluated under separate ownership categories.
    3. The term “default” means an act or omission by an insured depository institution which has been adjudged to constitute a breach of contract with the Corporation.
  2. Section 3(b) — Standard maximum deposit insurance amount. The term “standard maximum deposit insurance amount” means $250,000.

Task lists

Markdown-based CMS tools (Notion, GitHub, Contentful) generate checkbox lists for task tracking. These render as unstyled lists with <input type="checkbox" disabled> elements, using the brand blue as the accent color:

  • Verify institution CERT number and insurance status
  • Calculate insured deposits per ownership category
  • Prepare depositor notification letters
  • Coordinate with acquiring institution for assumption
  • Publish final receivership report

Definition lists

Definition lists use asymmetric spacing — more space above a term than below it — so each term groups visually with its own definition rather than with the previous entry:

Deposit Insurance
Protection provided by the FDIC that guarantees depositors will receive their insured funds if an insured bank or savings institution fails. The standard insurance amount is $250,000 per depositor, per insured bank, for each account ownership category.
Call Report
A quarterly financial report filed by every insured bank with its primary federal regulator. Call Reports provide detailed balance sheet, income statement, and off-balance-sheet data used for supervisory and analytical purposes.
Capital Adequacy
A measure of a bank's financial strength, typically expressed as a ratio of capital to risk-weighted assets. Regulators use capital adequacy ratios to determine whether a bank holds enough reserves to absorb potential losses.
Receivership
The process by which the FDIC manages and resolves a failed insured depository institution. The FDIC is appointed as receiver, takes control of the bank's assets, and works to maximize recoveries for depositors and creditors.

Multiple definitions per term

A single term can have multiple definitions — common in glossaries where a term has distinct meanings in different contexts. Use multiple <dd> elements after a single <dt>:

Resolution
The process of closing a failed bank and disposing of its assets, either through a purchase-and-assumption transaction or a deposit payoff.
A formal decision or statement adopted by a board of directors, such as a resolution authorizing a specific action or policy change.
Examination
An on-site review of a bank’s financial condition, risk management practices, and compliance with laws and regulations, conducted by federal or state examiners.
In legal proceedings, the questioning of a witness under oath, as in direct examination or cross-examination.

Tables

Tables use the brand blue for header backgrounds with white text. Body rows alternate with subtle striping, and a hover highlight aids row tracking. The <caption> element provides an accessible title, and <tfoot> contains summary rows.

Accessibility: Tables are wrapped in a <div class="prose-table-wrapper"> with role="region", a descriptive aria-label, and tabindex="0". This pattern preserves native table semantics for screen readers (row/column navigation) while allowing keyboard users to scroll wide tables. Never apply display: block to a <table> element — it destroys the accessibility tree. Use scope="col" on column headers and scope="row" on row headers so screen readers announce headers when navigating cells. The <caption> element provides a programmatic name for the table, announced when a screen reader enters the table context.

Column alignment

Choose alignment based on what the column contains. The goal is to make values easy to scan and compare vertically:

  • Left-align text, names, descriptions, and categories — anything readers scan as prose. This is the default.
  • Right-align numbers, currency, and percentages using class="prose-td-numeric". Right alignment keeps decimal points and digit places lined up so readers can compare magnitudes at a glance. Pair with font-variant-numeric: tabular-nums (already applied by the class) to ensure uniform digit widths.
  • Center-align short, categorical values like status badges, yes/no flags, or single-digit counts where vertical scanning isn’t meaningful.
  • Match header to data — always align the <th> the same way as the <td> cells beneath it so the label sits directly above the values it describes.

Components

Callouts

Callouts draw attention to important information without interrupting the reading flow. They use Phosphor icons and color variants derived from the design system's token palette:

Tip: You can verify your deposit insurance coverage using the FDIC’s Electronic Deposit Insurance Estimator (EDIE) tool at fdic.gov. It calculates coverage across all ownership categories for any FDIC-insured institution.

Coverage categories: Insured deposits include checking accounts, savings accounts, money market deposit accounts, and certificates of deposit (CDs) at FDIC-insured banks. Each ownership category — single accounts, joint accounts, revocable trusts, and retirement accounts — is insured separately up to $250,000.

Not insured: Deposit insurance does not cover investments such as stocks, bonds, mutual funds, annuities, or life insurance policies — even if they were purchased at an FDIC-insured bank or through an affiliated broker.

Your deposits are safe. Since the FDIC was established in 1933, no depositor has lost a single penny of insured funds. The Deposit Insurance Fund is backed by the full faith and credit of the United States government.

Bank closure notice: If your bank has been closed by regulators, contact the FDIC at (877) 275-3342 immediately. Insured depositors typically receive access to their funds within two business days of closure. Do not send deposits to a closed institution.

Accessibility: Callouts use role="note" to identify them as supplementary information. The danger variant uses role="alert" so screen readers announce it immediately (ARIA live region semantics). Each callout carries an aria-label matching its variant (e.g., “Information,” “Warning”) and the decorative icon span uses aria-hidden="true" to prevent the SVG data URI from being announced. Always include these ARIA attributes in callout markup:

<div class="prose-callout prose-callout-info" role="note" aria-label="Information">
  <span class="prose-callout-icon" aria-hidden="true"></span>
  <div class="prose-callout-content">
    <p>Your callout text here.</p>
  </div>
</div>

Progress bars

The <progress> element represents the completion of a task. Always pair it with a visible label and use aria-label or a <label> element for screen readers. When a value attribute is present, the bar fills to that proportion. Omitting value creates an indeterminate progress bar — a striped animation indicating work in progress without a known completion percentage:

Meters

The <meter> element represents a scalar measurement within a known range — ideal for capital ratios, compliance scores, or any bounded value. The low, high, and optimum attributes let the browser color the bar to indicate whether the value is acceptable:

Output element

The <output> element represents the result of a calculation or user action. It renders in a monospace font with a subtle background, similar to inline code but semantically distinct. For example, based on three beneficiaries at $250,000 each, the maximum insured amount is $750,000. Use <output> for computed values, conversion results, or system-generated responses within flowing text.

Collapsible sections

The <details> and <summary> elements create accordion-style collapsible sections. The summary uses a Phosphor chevron icon on the right side that rotates on open, with hover, active, and focus-visible states:

Accessibility: The <details>/<summary> pattern is natively keyboard-accessible — users can toggle with Enter or Space. Screen readers announce the summary as a button with an expanded/collapsed state. No additional ARIA is needed. The :focus-visible style provides a visible focus ring matching the design system’s standard outline pattern. In print, all details are force-expanded so content is not hidden.

What is deposit insurance?

Deposit insurance is a guarantee provided by the FDIC that protects depositors against the loss of their insured deposits if an FDIC-insured bank or savings association fails. Since the FDIC was established in 1933, no depositor has lost a single penny of insured funds.

How are bank examinations conducted?

Bank examinations are conducted on-site by teams of trained examiners who review a bank's financial condition, management practices, compliance with laws and regulations, and risk management processes. The examination results in a confidential report and a composite rating under the CAMELS system.

What happens when a bank fails?

When a bank fails, the FDIC is typically appointed as receiver. It assumes responsibility for selling or collecting the assets of the failed bank and settling its debts, including claims for deposits in excess of the insured limit. Insured depositors typically have access to their funds within two business days of the closure.

Footnotes

Footnotes use the standard scholarly pattern with bidirectional linking: a superscript reference[3] in the body links down to the footnote, and a back-link (↩) in the footnote returns to the reference. Clicking a footnote highlights it briefly with the :target pseudo-class. The footnote section uses a partial-width horizontal rule as a visual separator[4]:

Accessibility: Footnotes use DPUB-ARIA roles for assistive technology. Inline references use role="doc-noteref" so screen readers announce them as note references, not generic links. The footnote section uses role="doc-endnotes" with an aria-label, each <li> uses role="doc-footnote", and back-links use role="doc-backlink" with a descriptive title. The :target flash animation (yellow highlight) respects prefers-reduced-motion by falling back to a static highlight. Always include these roles in footnote markup:

<!-- Inline reference -->
<sup><a href="#fn1" id="ref1" role="doc-noteref">[1]</a></sup>

<!-- Footnote section -->
<section class="prose-footnotes" role="doc-endnotes" aria-label="Footnotes">
  <hr />
  <ol>
    <li id="fn1" role="doc-footnote">
      Footnote text.
      <a href="#ref1" role="doc-backlink" title="Back to reference">&#x21a9;</a>
    </li>
  </ol>
</section>

Accessibility

The prose component is designed for WCAG 2.1 AA conformance and Section 508 compliance. This section summarizes the accessibility patterns used throughout.

Keyboard access

Every interactive element in the prose component is reachable and operable via keyboard alone:

  • Skip link: The first element in <body> is <a href="#main" class="skip-link sr-only">Skip to content</a>. It becomes visible on focus, allowing keyboard users to bypass navigation and jump directly to the main content area.
  • Focus indicators: All focusable elements use :focus-visible with a 2px solid blue outline at 2px offset. This meets WCAG 2.4.7 (Focus Visible). The :focus-visible pseudo-class ensures outlines appear only for keyboard navigation, not mouse clicks.
  • Scrollable regions: Table wrappers and code blocks use tabindex="0" so keyboard users can scroll overflowing content with arrow keys. Each scrollable region has role="region" and an aria-label so screen readers announce entry into a scrollable area.
  • Details/summary: Natively keyboard-operable — Enter or Space toggles the expanded state.

Screen reader support

Semantic HTML and ARIA roles provide a clear experience for assistive technology users:

role="note" / role="alert"
Callouts use role="note" for informational content. The danger callout uses role="alert" to ensure immediate announcement by screen readers (ARIA live region semantics).
role="doc-noteref", role="doc-footnote", role="doc-backlink"
DPUB-ARIA roles on footnote references, footnote items, and back-links help screen readers convey the bidirectional footnote relationship. The wrapping <section> uses role="doc-endnotes" with an aria-label.
aria-hidden="true"
Decorative elements (callout icons, summary chevrons) are hidden from the accessibility tree. SVG data URIs embedded in CSS background-image are inherently invisible to screen readers; aria-hidden on icon <span> elements provides an explicit guarantee.
.sr-only utility
Visually hidden text provides screen reader context where visual styling alone conveys meaning. Used inside <del> and <ins> elements (deleted: / inserted:) since screen readers do not reliably announce these semantics. The clip-rect pattern (position: absolute; width: 1px; height: 1px; clip: rect(0,0,0,0)) is used rather than display: none, which would hide content from assistive technology entirely.
Table semantics
Tables use <caption> for programmatic names, scope="col" and scope="row" on header cells, and are never set to display: block. This preserves screen reader table navigation (cell-by-cell with header announcements).

Reduced motion

All transitions and animations are suppressed when the user has enabled prefers-reduced-motion: reduce in their operating system. This includes link underline transitions, table row hover effects, details chevron rotation, the footnote :target flash, and indeterminate progress bar animation. Where an animation conveys information (such as the footnote highlight), a static visual equivalent is used instead.

High contrast mode

A @media (forced-colors: active) block ensures the prose component works in Windows High Contrast mode (and other forced-color environments). Borders fall back to system colors (LinkText, ButtonText), elements that rely on color to convey meaning use forced-color-adjust: none, and <ins> elements fall back to an underline since background colors are overridden.

Content authoring checklist

Beyond the CSS and markup patterns above, accessible prose content requires attention from authors:

  1. Heading hierarchy: Use headings in order (h1 → h2 → h3) without skipping levels. Screen readers use heading levels to build a document outline for navigation.
  2. Descriptive link text: Write links that make sense out of context. “Read the quarterly report” is accessible; “click here” is not (WCAG 2.4.4).
  3. Alt text on images: Describe the image’s purpose, not its appearance. Use alt="" for decorative images.
  4. First-use abbreviation expansion: Spell out acronyms on first use: “Federal Deposit Insurance Corporation (FDIC).” The title attribute is not a substitute — many screen readers do not announce it by default.
  5. Table captions and headers: Every data table needs a <caption> and properly scoped <th> elements. Never use tables for layout.
  6. Language attribute: The lang="en" attribute on <html> enables correct screen reader pronunciation. Mark up foreign-language passages with lang on the containing element.
  7. Color independence: Never use color as the sole means of conveying information (WCAG 1.4.1). Callouts pair color with icons and bold lead text; table headers pair color with position and weight.
